But du projet

Votre application Android

Vous allez mettre en place votre propre application Android. Celle-ci doit rencontrer les critères suivants:

  • Modèle objet avec
    • une entité pour représenter les utilisateurs
    • au moins 2 autres entités de données
    • un champ identifiant pour chaque entité
  • Une application avec au minimum (en mode portrait) :
    • Une activité d'inscription / connexion
    • Une activité montrant une liste d'objets
    • Une activité montrant le détail d'un objet
    • Une interface permettant la création / saisie d'objets

Phase 1

Pendant la phase 1, vous devez

  • Définir les activités de votre application
  • Mise en page
  • Modèle
  • Service retrofit2
  • Fournir une implémentation mock de ce service
  • Navigation avec Drawer

4 points Activités et mise en page

Activités

Vous devez programmer les activités suivantes au minimum:

  • Écran de connexion et inscription
  • Écran d'accueil et d'affichage de liste
  • Écran de consultation d'un objet en détail
  • Interface (activité, dialogue etc.) de création d'objet
Pointage
  • 4 points 4 activités existent et sont mises en page

6 points Modèle

Besoins de modélisation

Vous devez modéliser au minimum les utilisateurs (au moins un identifiant et un mot de passe) ainsi qu'une entité que vous gérez (selon votre application). Cette entité doit représenter au moins 3 champs de données en plus de l'identifiant.

Vous devez modéliser tous les besoins qui découlent de votre interface graphique. Si le professeur identifie des informations non représentées ou représentées en double, vous perdrez des points.

Pointage
  • 2 points modélisation minimum atteinte
  • 4 points pas d'oublis ou de doublons

6 points Service et Retrofit

Identification du service

Vous devez identifier les différentes actions permises par votre service en fonction des écrans prévus dans votre application. Chaque bouton ou affichage de données est a priori un besoin en terme de service.

Vous devez atteindre le minimum soit la gestion des utilisateurs (création de compte, connexion et déconnexion) ainsi que la gestion d'une entité de données (création, récupération d'une liste, récupération d'une entité).

Vous devez déterminer toutes les méthodes qui découlent de votre interface graphique. Si le professeur identifie des méthodes manquantes, vous perdrez des points.

Pointage
  • 2 points service minimum atteint
  • 4 points pas d'oublis de méthodes

6 points Mock retrofit

Prototype fonctionnel sans serveur

Pour chaque méthode du service retrofit, vous devez fournir une implantation moquée. Il n'est pas nécessaire que l'information soit cohérente. Les objets retournés doivent uniquement servir à permettre la démonstration de l'interface.

Vous ne devez pas "hardcoder" d'information ailleurs que dans le mock. Chaque endroit dans le code d'interface qui ne fait pas appel au mock vous coutera un point.

Pointage
  • 2 points le service moqué est implanté
  • 4 points Le service est utilisé partout dans l'interface

7 points Navigation entre activités

Accès partout de partout

Vous devez avoir un drawer de navigation dans toutes les activités appropriées (cela n'a pas de sens pour l'écran de connexion par exemple). Le tiroir doit donner accès à tous les éléments de l'interface appropriés (au minimum, à l'accueil et à la déconnexion).

Vous devez vous assurer d'avoir le "hamburger" fonctionnel.

Vous devez faire une mise en page personnalisée (qui vous identifie) pour l'en-tête du tiroir (drawer header)

Vous devez vous assurer d'implanter les autres possiblités de navigation. Par exemple, quand un utilisateur se connecte, aller à l'écran d'accueil, quand l'utilisateur, sélectionne un item dans une liste aller à l'écran de détail, retourner à la liste après la création d'un objet etc.

Pointage
  • 3 points le tiroir est fonctionnel dans toutes les activités et donne accès à toute l'interface
  • 1 point tiroir : hamburger fonctionnel dans toutes les activités
  • 1 point tiroir : drawer header personnalisé
  • 2 points toutes autres navigations fonctionnelles
